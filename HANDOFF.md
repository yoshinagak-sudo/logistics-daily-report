# 物流日報AIアプリ - 引き継ぎ

最終更新: 2026-04-22

## プロジェクト概要
物流会社向けに外販する「AI運行日報アプリ」。ドライバーは「話す・撮る・確認する」だけで日報を完成、管理者はリアルタイムで運行・異常・労務リスクを把握できる。

## 現在の状態
- Next.js 16.2.4 (App Router) + TypeScript + Tailwind
- Supabase Client / Gemini SDK インストール済み
- **デモモード（localStorage）でフル機能が動作**

### 実装済み（法令対応含む）
**ドライバー画面（1画面スクロール）:**
- サマリー（配送/走行/拘束 + 改善基準告示の警告バッジ）
- 車両選択
- 出庫前 点呼: アルコールチェック（mg/L入力 + 検知器使用 + 確認方法 + 健康状態 + 睡眠時間 + 疲労度）
- 始業前点検: 法定10項目のチェックリスト + OBD警告灯 + 冷凍車温度
- タイムスタンプ（出勤/出庫/帰庫/退勤）
- 休憩開始/終了
- 配送実績（積載品目つき）
- メーターOCR（出庫時/帰庫時）
- 経費（給油: 量・金額、高速: 料金）
- 異常報告（種別/重要度/詳細）
- 退勤前 点呼（アルコールチェック）
- 音声入力 + 提出（画面下部固定）

**管理ダッシュボード:**
- サマリー6カード（提出率/配送/走行/異常/労務違反/アルコール要対応）
- AI要約
- ドライバー別運行テーブル（アバター + 提出/異常Pill）
- 詳細モーダル: 労務遵守 + 点呼 + 点検 + 配送積載品目 を表示
- CSV出力: 27カラム（点呼/点検/労務違反/積載/経費まで含む）

### 法令準拠の根拠
- アルコールチェック: 改正道路交通法（運送業 2011年〜、白ナンバー 2023年12月〜）
- 改善基準告示: 拘束13h/連続運転4h/休息11h（2024年4月適用）を自動判定
- 日常点検: 道路運送車両法第47条の2 + OBD点検（2021年10月〜）

## 未解決の課題・ハマりどころ
- **Supabase認証情報未設定**: `.env.local` に `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` が必要
- **Gemini API Key 未設定**: `GOOGLE_GENERATIVE_AI_API_KEY` が必要
- **音声録音の権限**: iOS Safari は HTTPS 必須、localhost は OK だが本番は Vercel HTTPS で動作確認必要
- **メーターOCR精度**: 数字確定はユーザー確認ステップを必ず挟む（誤認識による距離間違いを防ぐ）
- 古い実装 `/Desktop/system-dev/音声入力配送日報/` は無視（2026年2月の使い捨てプロト、日本語パスでビルドエラー懸念）
